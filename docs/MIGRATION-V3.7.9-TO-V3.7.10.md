# Migration: EDIS 3.7.9 to 3.7.10

## Scope

EDIS 3.7.10 is a validation-hardening patch release. It adds source-only validation tooling and controlled real-fixture intake contracts. It does not add a user-facing feature, change frozen evidence schemas, add ZIP64, or move Python/Browser responsibilities into WordPress.

## Runtime behavior

The installed plugin behavior remains contract-compatible with 3.7.9. The producer and implementation version are updated to `3.7.10`, so an incomplete private Job created by an older implementation must be recreated rather than silently resumed. Completed export packages are not rewritten.

## Source repository changes

- `tools/validation/run-local-validation.php` runs repository-owned local gates and records command, exit code, duration, output hashes, and evidence state.
- `tools/validation/run-local-validation.ps1` exposes the same runner on Windows PowerShell.
- `validation/validation-plan.json` records external gates without converting `NOT_RUN` or `BLOCKED_EXTERNAL` into PASS.
- `tests/fixtures/elementor-real/` defines fail-closed intake for controlled real Elementor fixtures. No synthetic fixture is labelled real.
- `validation/` and `tools/` remain excluded from the WordPress installation ZIP.

## Composer limitation

`composer.lock` is not generated by this package because the build environment did not have Composer dependency resolution available. Do not hand-author the lockfile. Generate it in a networked release environment with:

```bash
composer validate --strict
composer update --with-all-dependencies
composer install --no-interaction --prefer-dist
composer audit --locked
```

Commit the resulting lockfile only after review and successful official PHPUnit/PHPCS execution.

## Installation

Replace the existing plugin directory with the 3.7.10 installation ZIP. Keep the configured private evidence storage directory outside the public WordPress document root.
